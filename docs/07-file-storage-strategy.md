# راهبرد ذخیره‌سازی فایل‌ها

## هدف

پرونده‌های خرید دارای مدارک و پیوست‌های متعدد هستند. این فایل‌ها باید در یک Root Folder Repository نگهداری شوند و پایگاه داده فقط متادیتا و مسیر کنترل‌شده آن‌ها را ذخیره کند.

## اصل طراحی

- فایل فیزیکی در SQL Server ذخیره نمی‌شود.
- مسیر، نام فایل، نوع سند، اندازه، هش، تاریخ بارگذاری و شناسه پرونده در پایگاه داده ذخیره می‌شود.
- تولید مسیر فایل باید فقط از طریق یک سرویس مرکزی انجام شود.
- کاربران نباید مسیر فیزیکی فایل را به صورت مستقیم وارد کنند.

## ساختار پیشنهادی Root Folder

```text
RootFolder/
  PurchaseFiles/
    {PurchaseFileNumber-or-Id}/
      Incoming/
      Technical/
      Commercial/
      Tender/
      Warehouse/
      Final/
      Archive/
```

## نمونه مسیر

```text
RootFolder/PurchaseFiles/PF-2026-000123/Tender/meeting-minutes-v1.pdf
```

## متادیتای پیشنهادی فایل

در فاز پیاده‌سازی، برای هر فایل بهتر است اطلاعات زیر نگهداری شود:

- شناسه فایل
- شناسه پرونده خرید
- نوع سند
- نام اصلی فایل
- نام ذخیره‌شده فایل
- مسیر نسبی نسبت به Root Folder
- پسوند فایل
- نوع MIME
- اندازه فایل
- هش فایل برای کنترل تغییر یا تکرار
- کاربر بارگذاری‌کننده
- تاریخ بارگذاری
- نسخه سند
- وضعیت سند

## دسته‌بندی سندها

دسته‌بندی اولیه می‌تواند شامل موارد زیر باشد:

- Incoming: اسناد ورودی و درخواست اولیه
- Technical: مدارک فنی و توضیحات متقاضی
- Commercial: پیشنهادها و مدارک تجاری
- Tender: اسناد مناقصه و کمیسیون
- Warehouse: مدارک و نظرهای انبار
- Final: اسناد نهایی و رسمی
- Archive: اسناد آرشیوشده یا قدیمی

## سیاست نسخه‌بندی

اگر سندی اصلاح می‌شود، بهتر است نسخه قبلی حذف فیزیکی نشود. نسخه جدید با شماره نسخه بالاتر ذخیره شود و نسخه فعال در پایگاه داده مشخص باشد.

نمونه نام‌گذاری:

```text
technical-specification-v1.pdf
technical-specification-v2.pdf
```

## امنیت و دسترسی

- دانلود فایل باید از طریق API کنترل‌شده انجام شود.
- دسترسی به فایل باید بر اساس نقش، واحد و ارتباط با پرونده بررسی شود.
- مسیر فیزیکی Root Folder نباید در UI نمایش داده شود.
- عملیات حذف باید محدود، قابل ثبت و ترجیحاً نرم‌حذف باشد.

## ارتباط با AI Agent

در فازهای آینده، AI Agent برای خلاصه‌سازی و بررسی اسناد به فایل‌های پرونده نیاز دارد. بنابراین متادیتای فایل‌ها باید برای جستجو، طبقه‌بندی و استخراج متن آماده باشد.

موارد مهم برای آینده:

- ثبت نوع سند
- ثبت نسخه سند
- ثبت وضعیت قابل پردازش بودن فایل
- امکان استخراج متن و نگهداری خروجی استخراج‌شده
- آماده‌سازی برای ایندکس RAG
